# Yakov Viewer Project Preamble

## Product Identity

Yakov Viewer is the public photo portfolio and archive interface for Yakov Shmol.

It is not a generic gallery and not the private Logjamming tool. Logjamming remains a related backstage/future project, but the current product is a premium public portfolio with a serious archive foundation behind it.

## Current Priority

The project now has two equally important surfaces:

- a minimal, photo-first public portfolio;
- a private admin and storage workspace for uploading, reviewing, organizing, hiding, publishing, and deleting photographs.

The admin is not just a control panel for the public website. It is a private working archive. It must support albums and photos that may never be published.

## Long-Term Direction

The same archive database and R2 storage should eventually serve both:

- Yakov Viewer, the public portfolio;
- Logjamming, the future/private photo workflow and sorting project.

The user does not want to upload the same archive twice for two related products. Storage, metadata, album structure, tags, and permissions should be designed as shared infrastructure.

## Product Principles

- The photo is the main interface.
- Public UI should stay extremely minimal.
- Text, buttons, and metadata should not compete with images.
- Photos themselves should generally not be rounded.
- Admin can be dense and practical; public site should be quiet and immersive.
- Hidden is not delete.
- Publishing should be deliberate: upload, review, then publish.
- The system should prioritize quality and control over lowest possible storage cost.
